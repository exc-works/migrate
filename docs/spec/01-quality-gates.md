# 01 Quality Gates

## 目标

本分册定义独立项目必须满足的质量门禁，用于约束实现行为与回归风险。

## 必修复问题类型

1. out-of-order 行为准确性

- 当 `MigrateOutOfOrder=false` 时，out-of-order 版本必须跳过，不得执行。

2. 解析器错误处理

- 解析失败必须返回 `error`，不得导致进程退出。

3. 版本字段语义一致性

- 各数据库中 `version` 必须保持字符串语义，避免类型不一致。

4. 事务完整性

- `Exec/Commit/Rollback` 的错误必须可观测并返回。

5. 测试可复现性

- 测试不得依赖开发机本地固定数据库。

6. 文件系统权限正确性

- 目录和文件创建权限需符合跨平台预期。

7. 错误信息准确性

- 错误信息必须包含正确的变量名与上下文，不得输出误导内容。

## Gate

进入提交前，必须确认：

- 以上问题类型都有对应测试或验证步骤。
